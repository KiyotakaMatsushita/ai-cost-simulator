# AI コストシミュレーター 計算ロジック説明書

## 概要

本シミュレーターは、Gemini 2.5 Flash / Gemini 3.0 Flash / Claude Haiku 4.5 の3モデルにおいて、**コンテキストキャッシュを利用した場合**の運用コストを比較するツールです。

---

## 入力パラメーター

| 記号 | パラメーター名 | デフォルト値 | 説明 |
|------|---------------|-------------|------|
| C | 初期キャッシュ量 | 50,000 tokens | キャッシュに書き込むトークン数（システムプロンプト等） |
| R | 再利用ヒット量 | 50,000 tokens | 1リクエストあたりのキャッシュ読み出しトークン数 |
| P | 新規プロンプト | 7,000 tokens | 1リクエストあたりの新規入力トークン数 |
| O | 回答トークン | 2,000 tokens | 1リクエストあたりの出力トークン数 |
| U | 利用者数 | 2,500 人 | テスト対象のユーザー数 |
| N | 回数/ユーザー | 10 回 | 1ユーザーあたりのリクエスト回数 |
| D | テスト日数 | 7 日 | 運用期間 |
| h | Gemini保持時間 | 24 時間/日 | 1日あたりのキャッシュ保持時間（Geminiの保管料計算用） |

### 導出値

| 値 | 計算式 | 説明 |
|----|--------|------|
| 総リクエスト数 | `U × N` | 全ユーザーの合計リクエスト数 |
| 総保管時間 | `h × D` | キャッシュの累計保管時間 |
| スケール倍率 | 共有: `1` / 個別: `U` | キャッシュが共有か個別かで変わる |

### 課金モデルの違い

- **Gemini**: 書き込みは入力と同額（上乗せなし）。代わりに保管料が時間単位で発生。
- **Claude**: 書き込みに上乗せ（5分TTL: 1.25倍、1時間TTL: 2.0倍）。保管料は無料。TTL内にリクエストがあればキャッシュが自動延長される。

## 計算式（5段階）

### 1. キャッシュ作成費（Write Cost）

キャッシュを作成する際のコスト。日次でキャッシュを再作成する想定。

```
Write Cost = (C / 1,000,000) × WritePrice × D × ScaleMultiplier
```

- **Gemini の WritePrice** = Input単価（上乗せなし）
- **Claude の WritePrice** = 5分TTL: Input × 1.25 / 1時間TTL: Input × 2.0
- **ScaleMultiplier** = 共有キャッシュ: 1 / 個別キャッシュ: U

**例: Gemini 2.5 Flash（共有キャッシュ）**
```
= (50,000 / 1,000,000) × $0.30 × 7 × 1
= 0.05 × 0.30 × 7
= $0.105
```

### 2. 保管料（Storage Cost）

キャッシュを保持している時間に対する課金。**Geminiのみ発生、Claudeは無料。**

```
Storage Cost = (C / 1,000,000) × StoragePrice × (h × D) × ScaleMultiplier
```

**例: Gemini 2.5 Flash（共有キャッシュ、24h/日）**
```
= (50,000 / 1,000,000) × $1.00 × (24 × 7) × 1
= 0.05 × 1.00 × 168
= $8.40
```

### 3. キャッシュ再利用費（Read Cost）

各リクエストでキャッシュから読み出す際のコスト。入力単価の10%で大幅割引。

```
Read Cost = (R / 1,000,000) × CachedInputPrice × (U × N)
```

> ※ ScaleMultiplierは不要。各リクエストが1回ずつ読むため、総リクエスト数で計算。

**例: Gemini 2.5 Flash**
```
= (50,000 / 1,000,000) × $0.03 × (2,500 × 10)
= 0.05 × 0.03 × 25,000
= $37.50
```

### 4. 新規入力費（Prompt Cost）

キャッシュに含まれない新規プロンプト部分のコスト。通常入力単価で課金。

```
Prompt Cost = (P / 1,000,000) × InputPrice × (U × N)
```

**例: Gemini 2.5 Flash**
```
= (7,000 / 1,000,000) × $0.30 × 25,000
= 0.007 × 0.30 × 25,000
= $52.50
```

### 5. 出力費（Output Cost）

モデルの生成した回答トークンに対するコスト。キャッシュの有無に関係なく固定。

```
Output Cost = (O / 1,000,000) × OutputPrice × (U × N)
```

**例: Gemini 2.5 Flash**
```
= (2,000 / 1,000,000) × $2.50 × 25,000
= 0.002 × 2.50 × 25,000
= $125.00
```

## 合計コスト

```
Total Cost = Write Cost + Storage Cost + Read Cost + Prompt Cost + Output Cost
```

**例: Gemini 2.5 Flash（共有キャッシュ、24h保持、デフォルトパラメーター）**
```
= $0.105 + $8.40 + $37.50 + $52.50 + $125.00
= $223.505
```

### 1リクエストあたりのコスト

```
Per Request Cost = Total Cost / (U × N)
```

```
= $223.505 / 25,000
= $0.00894 ≈ ¥1.39（@¥155/USD）
```

## キャッシュなしとの比較（削減率の計算）

キャッシュを使わない場合、キャッシュ読み出し分（R）もすべて通常入力単価で課金されます。

```
No-Cache Cost = ((R + P) / 1,000,000) × InputPrice × (U × N) + Output Cost
```

```
削減額 = No-Cache Cost - Total Cost
削減率 = (削減額 / No-Cache Cost) × 100
```

**例: Gemini 2.5 Flash**
```
No-Cache Cost = ((50,000 + 7,000) / 1,000,000) × $0.30 × 25,000 + $125.00
             = 0.057 × 0.30 × 25,000 + 125.00
             = $427.50 + $125.00
             = $552.50

削減額 = $552.50 - $223.505 = $328.995
削減率 = ($328.995 / $552.50) × 100 = 59.5%
```

## 共有キャッシュ vs 個別キャッシュ

### 共有キャッシュ（Shared）
- 全ユーザーで1つのキャッシュを共有（例：会社のマニュアル、システムプロンプト）
- ScaleMultiplier = **1**
- Write Cost と Storage Cost はユーザー数に依存しない
- **Gemini有利**: 保管料が固定のため、ユーザー数が多いほど1人あたりのコストが下がる

### 個別キャッシュ（Personalized）
- ユーザーごとに専用キャッシュを作成（例：個人の会話履歴、専用ドキュメント）
- ScaleMultiplier = **U（ユーザー数）**
- Write Cost と Storage Cost がユーザー数に比例して増加
- **Claude有利**: 保管料が無料のため、個別キャッシュが大量にあってもStorageコストが発生しない

## まとめ

| 観点 | Gemini | Claude |
|------|--------|--------|
| 書き込み | 入力と同額（安い） | 1.25〜2.0倍の上乗せ |
| 保管料 | 時間課金（長時間保持で高額化） | 無料 |
| 再利用 | 入力の10%（安い） | 入力の10%（同率だが基本単価が高い） |
| 共有キャッシュ | 非常に有利 | 普通 |
| 個別キャッシュ | 保管料がU倍で高額化 | 保管料無料で有利 |

**結論**: ユースケース（共有 or 個別）とキャッシュ保持時間によって最適なモデルが変わります。シミュレーターで各パラメーターを変更して比較してください。
